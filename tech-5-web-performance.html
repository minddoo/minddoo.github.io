1. Core Web Vitals의 이해

구글의 Core Web Vitals는 웹사이트의 사용자 경험(UX)을 측정하는 핵심 지표로, 검색 엔진 최적화(SEO) 순위에 직접적인 영향을 미칩니다. 주요 지표로는 최대 콘텐츠 풀 페인트(LCP), 최초 입력 지연(FID)을 대체한 상호작용성 지표(INP), 그리고 누적 레이아웃 이동(CLS)이 있습니다. 이 세 가지 지표를 모두 녹색(Good) 등급으로 유지하는 것은 단순한 성능 최적화를 넘어 비즈니스의 전환율과 직결되는 매우 중요한 과제입니다. 사용자는 로딩이 3초 이상 지연되면 50% 이상이 이탈하기 때문입니다.

2. LCP 최적화: 이미지 렌더링과 리소스 우선순위

화면에서 가장 큰 요소를 렌더링하는 데 걸리는 시간인 LCP를 단축하려면 초기 로딩 리소스의 최적화가 필수적입니다. 가장 흔한 원인은 용량이 큰 히어로 이미지나 웹 폰트입니다. 이를 해결하기 위해 차세대 이미지 포맷인 WebP나 AVIF를 사용하고, `srcset` 속성을 통해 디바이스 해상도에 맞는 이미지를 제공해야 합니다. 또한 중요한 이미지에는 ``를 설정하고 `fetchpriority="high"` 속성을 주어 브라우저가 최우선으로 다운로드하도록 지시하는 기법이 필수적으로 요구됩니다.

3. INP와 자바스크립트 실행 최적화

최근 도입된 INP(Interaction to Next Paint)는 사용자의 클릭, 타이핑 등 모든 상호작용의 지연 시간을 측정합니다. INP 불량의 주된 원인은 브라우저의 메인 스레드(Main Thread)를 길게 점유하는 무거운 자바스크립트 실행(Long Tasks)입니다. 이를 해결하기 위해서는 코드를 적절히 분할(Code Splitting)하고, Web Worker를 활용해 무거운 연산을 백그라운드 스레드로 위임해야 합니다. React 환경에서는 `useTransition`이나 `useDeferredValue` 훅을 활용하여 중요도가 낮은 렌더링의 우선순위를 미루는 동시성 렌더링(Concurrent Rendering) 패턴이 적극 활용됩니다.

4. CLS 방어: 안정적인 레이아웃 구조 설계

웹 페이지가 로딩되는 도중 요소들이 갑자기 밀려나는 현상(CLS)은 사용자에게 큰 불쾌감을 줍니다. CLS를 방지하기 위해 이미지나 광고 박스, 비디오 등 크기를 동적으로 결정하는 모든 미디어 요소에 `width`와 `height` 속성을 명시하거나 CSS `aspect-ratio` 속성으로 스켈레톤 UI를 미리 확보해야 합니다. 또한 커스텀 웹 폰트가 로드되면서 발생하는 FOIT/FOUT 현상으로 인한 텍스트 흔들림을 막기 위해 `font-display: swap` 옵션과 시스템 폰트 폴백(Fallback)을 적절히 구성해야 합니다.
